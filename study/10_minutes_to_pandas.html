# Criar dataframe usando dicionario 
'''
Cada chave no dicionário representa o nome de uma coluna do DataFrame e os valores correspondentes representam os dados para aquela coluna.
As colunas do DataFrame df2 são as seguintes:
A: Uma coluna com o valor constante 1.0.
B: Uma coluna com um objeto Timestamp representando a data "20130102".
C: Uma coluna que contém uma série com valores 1.0, indexados de 0 a 3.
D: Uma coluna que contém um array NumPy com quatro elementos, todos com o valor 3, com tipo de dados "int32".
E: Uma coluna que contém dados categóricos (categorias) representados por objetos Categorical. Os valores são ["test", "train", "test", "train"].
F: Uma coluna com o valor constante "foo".
'''
df2 = pd.DataFrame(
    {
        "A" : 1.0,
        "B" : pd.Timestamp("20130102"),
        "C" : pd.Series(1,index= list(range(4)), dtype = "float32"),
        "D" : np.array([3]*4, dtype = "int32"),
        "E" : pd.Categorical(["test", "train", "test", "train"]),
        "F" : "foo"
    }
)

print(df2)

NOTA: DataFrame.to_numpy() fornece uma representação NumPy dos dados subjacentes. Observe que esta pode ser uma operação cara quando seu DataFrame possui colunas com diferentes tipos de dados, o que se resume a uma diferença fundamental entre pandas e NumPy: os arrays NumPy têm um dtype para todo o array, enquanto os pandas DataFrames têm um dtype por coluna. Quando você chama DataFrame.to_numpy(), os pandas encontrarão o NumPy dtype que pode conter todos os dtypes no DataFrame. Isso pode acabar sendo objeto, o que requer a conversão de todos os valores para um objeto Python. '''

#Operating with objects that have different dimensionality and need alignment. 
#In addition, pandas automatically broadcasts along the specified dimension:
'''
cria uma nova Series chamada s usando o construtor pd.Series(). 
Essa Series é construída a partir de uma lista de valores [1, 3, 5, np.nan, 6, 8] e um índice definido como dates. 
Em seguida, o método shift(2) é aplicado na Series resultante.
O método shift() é usado para deslocar (mover) os valores de uma Series para cima ou para baixo ao longo do índice. 
Nesse caso, shift(2) desloca os valores da Series duas posições para baixo ao longo do índice. 
Isso significa que os dois primeiros valores da Series são preenchidos com NaN (valores ausentes), 
e os demais valores são movidos para baixo duas posições.
'''
s = pd.Series([1, 3, 5, np.nan, 6, 8], index=dates).shift(2)
print(s)

'''

HDF5

Um arquivo HDF5 (Hierarchical Data Format 5) é um formato de arquivo de dados binários usado para armazenar grandes volumes de dados de forma eficiente. O HDF5 é um formato flexível, portátil e extensível que permite organizar, armazenar e gerenciar uma variedade de tipos de dados, incluindo conjuntos de dados multidimensionais, estruturas hierárquicas e metadados.

O formato HDF5 é amplamente utilizado em áreas que exigem armazenamento eficiente de dados científicos, como física, astronomia, bioinformática, finanças e ciência dos materiais. Ele é capaz de lidar com grandes conjuntos de dados e oferece suporte a recursos como compressão, filtragem, armazenamento de metadados detalhados e acesso eficiente a dados através de índices.

Um arquivo HDF5 pode conter vários conjuntos de dados, chamados de grupos, organizados em uma estrutura de árvore hierárquica semelhante a um sistema de arquivos. Cada grupo pode conter datasets (conjuntos de dados) que armazenam os próprios dados. Os datasets podem ser multidimensionais, como arrays, ou unidimensionais, como séries temporais.

O formato HDF5 também suporta atributos, que são metadados associados a grupos e datasets. Os atributos podem conter informações adicionais sobre os dados, como unidades, descrições, autores, entre outros.

Os arquivos HDF5 podem ser acessados e manipulados usando bibliotecas em várias linguagens de programação, incluindo Python, MATLAB, Java e C/C++. No contexto do pandas em Python, a biblioteca pytables é usada para trabalhar com arquivos HDF5
